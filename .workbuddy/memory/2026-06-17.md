## 移动端 / 电脑端平台分离

### JS 改动（game.js）
- 新增 `cachedIsMobile` + `IS_MOBILE_CACHE_MS = 300`，`isMobileInput()` 内部缓存结果，300ms 内复用，避免每帧构造 matchMedia
- 新增 `detectPlatform()` + `currentPlatform` + `onPlatformChange()`，统一管理平台状态
- 新增 `invalidateMobileCache()`，在 resize / orientationchange 时清除缓存并重新判定
- `updateMobileControls()` 改写为：桌面端直接 return（不操作 DOM），移动端才执行显示/隐藏逻辑
- 从 `updateMap()` 中移除每帧调用的 `updateMobileControls()`，改为在状态切换/resize/平台变更时调用
- `hideVirtualControls()` 改写为桌面端直接 return
- `showStartScreen()` / `hideStartScreen()` 中加入 `detectPlatform()` 调用
- resize / orientationchange 事件：加入 `invalidateMobileCache()` + `detectPlatform()` + 仅游戏中进行 `updateMap()`
- 初始化流程改为 `detectPlatform()` → `resizeCamera()` → `requestAnimationFrame`
- `touchmove` 事件监听器改用 `currentPlatform === "desktop"` 判断，替代 `isMobileInput()` 调用

### HTML 改动（index.html）
- `control-bar` 加上 `desktop-only` 类，配合 CSS 在移动端彻底隐藏
- `mobilePauseButton` 加上 `mobile-only` 类
- `mobileControls` 加上 `mobile-only` 类

### CSS 改动（style.css）
- 修复 `.meter` 重复定义（原 339 行和 1100 行）：将 `touch-action: manipulation` 合并到唯一一份 `.meter` 中
- 在移动端 media query（`@media (max-width: 768px), (hover: none) and (pointer: coarse)`）中：
  - 加入 `.desktop-only, .control-bar { display: none !important; }`（原 1202 行附近已包含，确认生效）
  - 加入 `.mobile-only { display: inline-flex; }`，确保移动端显示
  - 加入 `.mobile-controls { display: grid; }`，确保移动端虚拟摇杆容器使用 grid 布局
- 整理后 `.mobile-only` 在桌面端默认 `display: none`（305 行），在移动端 media query 中覆盖为 `display: inline-flex`
